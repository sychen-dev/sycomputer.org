# HeroSection 按鈕重疊問題解決方案

## 問題描述

HeroSection 中的「立即諮詢」按鈕（`button.consultNow`）和「探索更多」滾動指示器（`common.exploreMore`）在視覺上發生重疊和位置衝突。

### 問題分析
1. **視覺重疊**：兩個按鈕在螢幕上實際重疊在一起，導致文字或背景互相遮擋
2. **位置衝突**：兩個按鈕在垂直或水平位置上太靠近，影響美觀或可用性
3. **可能原因**：最近移除了「Learn More」按鈕，改變了垂直空間分配

## 解決方案選擇

**選擇方案**：垂直間距調整（選項1）

**理由**：
- 最簡單且風險最低
- 保持現有設計的一致性
- 可以快速實施和測試
- 如果無效，可以再嘗試其他選項

## 設計方案

### 具體修改

#### 1. 調整「立即諮詢」按鈕的下方間距
- **當前**：`mb-16`（約4rem/64px）
- **建議**：調整為`mb-24`（6rem/96px）或`mb-32`（8rem/128px）
- **目的**：增加按鈕與底部滾動指示器之間的空間

#### 2. 調整「探索更多」指示器的位置
- **當前**：`bottom-12`（約3rem/48px）
- **建議**：調整為`bottom-8`（2rem/32px）或`bottom-10`（2.5rem/40px）
- **目的**：使滾動指示器稍微上移，避免與按鈕重疊

#### 3. 可選：響應式調整
- 針對不同螢幕尺寸進行微調
- 例如：在小螢幕上使用更大的間距

### 修改位置

**檔案**：`components/sections/HeroSection.tsx`

**第101行**（按鈕容器）：
```tsx
// 當前
className="flex justify-center mb-16"

// 建議
className="flex justify-center mb-24"  // 或 mb-32
```

**第120行**（滾動指示器容器）：
```tsx
// 當前
className="absolute bottom-12 left-1/2 transform -translate-x-1/2"

// 建議
className="absolute bottom-8 left-1/2 transform -translate-x-1/2"  // 或 bottom-10
```

## 預期結果

1. 兩個元素之間有足夠的視覺間距
2. 保持HeroSection的整體美觀
3. 避免任何形式的視覺重疊
4. 維持現有的動畫和互動效果

## 潛在風險

1. 可能需要針對不同螢幕尺寸進行測試和調整
2. 如果問題只在特定條件下發生，可能需要額外的診斷
3. 調整過大可能影響HeroSection的整體比例

## 測試計劃

### 視覺測試
1. 在各種螢幕尺寸上測試（移動端、平板、桌面）
2. 檢查不同內容高度下的顯示情況
3. 驗證無視覺重疊

### 功能測試
1. 確保所有按鈕仍然可點擊
2. 確認動畫效果正常
3. 測試滾動功能

## 替代方案

### 方案2：重新設計佈局
- 將「探索更多」指示器移到「立即諮詢」按鈕的右側或左側
- 使用水平排列而不是垂直堆疊
- **缺點**：需要較大的代碼修改

### 方案3：響應式調整
- 針對不同螢幕尺寸使用不同的佈局
- **缺點**：需要最複雜的CSS調整

## 實施時間估計

- **主要修改**：15-30分鐘
- **測試和調整**：30-60分鐘
- **總計**：45-90分鐘
